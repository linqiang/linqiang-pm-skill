# 海参卖家成交助手 — 微信小程序落地方案

## 痛点确认

基于分析，锁定海参卖家的核心痛点：

1. **重复讲解**：每个客户都要从头介绍产品、泡发方法、价格
2. **客户管理靠记忆**：谁买过什么、什么时候该复购，全凭脑子记
3. **转介绍没有载体**：老客户想推荐给朋友，没有一个可以直接转发的东西

## 产品定义：海参卖家成交助手（微信小程序）

### 第一版功能清单（MVP，2 周开发）

| 模块 | 功能 | 优先级 | 解决的痛点 |
|------|------|--------|-----------|
| 产品展示 | 海参品类卡片：图片、规格、价格、产地说明 | P0 | 不用每次重新发图讲解 |
| 产品展示 | 泡发教程（图文或短视频） | P0 | 不用每次重新教 |
| 产品展示 | 客户好评/实拍展示墙 | P1 | 建立信任，替代朋友圈翻记录 |
| 成交转化 | 一键微信联系卖家 | P0 | 保留私聊成交习惯 |
| 成交转化 | 小程序内直接下单 + 微信支付 | P1 | 减少手动收款对账 |
| 分享传播 | 产品页一键转发给微信好友/群 | P0 | 老客户转介绍有载体 |
| 分享传播 | 生成产品海报图保存到相册 | P2 | 朋友圈传播 |

### 第二版功能（验证通过后，第 2-3 个月）

| 模块 | 功能 | 触发条件 |
|------|------|---------|
| 客户管理 | 购买记录自动沉淀（谁买了什么、多少钱、什么时候） | MVP 下单功能跑通后 |
| 复购提醒 | 到季/到期自动推送（"您上次买的海参快吃完了"） | 有 50+ 客户数据后 |
| 会员体系 | 老客户专属价、积分兑换 | 复购率数据验证后 |
| 内容运营 | 海参知识专栏（养生搭配、鉴别真假） | 用户留存数据验证后 |

### 不做的事情（明确排除）

- 不做通用电商平台功能（购物车、满减、拼团）
- 不做直播（海参卖家的成交场景不在直播）
- 不做多商家入驻（只服务单个卖家）
- 不做社区/论坛

## 技术方案

### 技术选型

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | 微信小程序原生（WXML + WXSS） | 生态最成熟，转发分享体验最好 |
| 后端 | 微信云开发（CloudBase） | 免服务器、自带微信支付能力、个人也能用 |
| 数据库 | 云开发内置 MongoDB | 够用，零运维 |
| 存储 | 云开发云存储 | 图片/视频存储，自带 CDN |
| 支付 | 微信支付（小程序内） | 原生集成，用户体验最好 |

## 落地排期

### 第 1 周：基础搭建

| 天 | 任务 |
|----|------|
| Day 1-2 | 小程序注册、云开发环境搭建、数据库集合创建、基础项目结构 |
| Day 3-4 | 首页 + 产品列表页开发，产品数据录入 |
| Day 5 | 产品详情页（含泡发教程展示） |

### 第 2 周：成交闭环

| 天 | 任务 |
|----|------|
| Day 6-7 | 下单流程 + 微信支付对接 |
| Day 8 | 分享功能（转发好友/群 + 海报生成） |
| Day 9 | 联系卖家（跳转微信客服）、好评展示墙 |
| Day 10 | 联调测试、真机调试、提交审核 |

### 第 3-4 周：验证

| 任务 | 具体内容 |
|------|---------|
| 种子用户 | 找 3-5 个海参卖家试用 |
| 数据埋点 | 分享次数、详情页浏览时长、下单转化率 |
| 用户访谈 | 每周和卖家聊 15 分钟：用了吗？哪里别扭？ |

## 成本估算

| 项目 | 费用 |
|------|------|
| 微信小程序认证 | 300 元/年 |
| 云开发基础版 | 免费（个人额度够 MVP 用） |
| 微信支付商户号 | 免费开通，0.6% 交易手续费 |
| 域名（可选） | 约 50 元/年 |
| **总计** | **约 350 元 + 交易手续费** |

## 验证成功标准

| 指标 | 目标值 | 意义 |
|------|--------|------|
| 卖家日均分享次数 | ≥ 3 次 | 说明小程序替代了重复讲解 |
| 客户通过小程序咨询占比 | ≥ 30% | 说明客户愿意先看再问 |
| 详情页平均浏览时长 | ≥ 40 秒 | 说明内容有吸引力 |
| 首月通过小程序成交订单 | ≥ 10 单 | 说明成交链路跑通 |
