# Nan Yu PM Skill

以 Linear 产品负责人 Nan Yu 的方法论为内核，为 [Claude Code](https://docs.anthropic.com/en/docs/claude-code) 打造的产品经理 Skill。帮助专业产品经理在产品全生命周期中做出更好的决策——从 0 到 1 构思、可行性验证、迭代优化，到上线复盘。

## 创建思路

### 为什么做这个 Skill

市面上的 AI 产品助手大多停留在"帮你写 PRD"或"生成用户故事"的层面。它们能帮你格式化文档，但无法帮你做**产品判断**。

这个 Skill 的出发点不同：它不是一个文档生成器，而是一个**产品思维伙伴**。它的核心价值是帮你在模糊的需求中找到真正的用户痛点，在多个可能的方向中做出有依据的取舍。

### 方法论来源

Skill 的方法论内核来自 Nan Yu（Linear 产品负责人）的产品哲学，核心理念包括：

1. **痛点挖掘**：不问"你要什么"，追问"你哪里难受"。通过表层需求 → 行为真相 → 情感内核三层递进，找到用户真正的痛苦
2. **先试极端，再折中**：不从"合理方案"出发，先推到两个极端，再在中间找到用户价值最大、成本可控的平衡点
3. **无意识苦差事**：找到用户每天在做、但从未抱怨过的重复劳动——这些往往是最高价值的产品机会
4. **快速验证**：最小可用版本优先，假设驱动，一周内完成构建 → 测量 → 学习的闭环
5. **PM 即 GTM**：每个产品决策都应回答——上线后用户怎么发现它、理解它、告诉别人
6. **结构化复盘**：对假设诚实，用数据验证、提炼意外发现、落地为加倍/调整/止损三类行动

### 设计原则

- **说人话**：像一个有经验的同事在聊天，不是在做咨询报告
- **给真实判断**：敢于说"这个方向可能不对"，而不是什么都说好
- **场景驱动**：用具体例子代替抽象概念，用落地方案代替空洞分析
- **模块化输出**：根据场景灵活组合输出模块，不必每次走完全流程

```

├── agents/
```

### 各文件说明

| 文件 | 作用 | 什么时候看 |
|------|------|-----------|
| `SKILL.md` | Skill 的入口，包含六大方法论摘要、七步对话流程、六个输出模块 | Claude Code 自动加载，你不需要手动打开 |
| `agents/interface.yaml` | 定义 agent 的角色、触发条件、排除场景、对话流程 | 想了解或修改触发规则时 |
| `references/methodology.md` | 方法论的完整操作指南，包含追问话术、极端推演步骤、复盘模板等 | 想深入理解每个方法论的具体用法时 |
| `examples/海参小程序落地方案.md` | 一个完整的使用示例 | 想看看实际产出长什么样时 |

## 使用方法

### 安装

```bash
npx skills@latest add linqiang/nan-yu-pm-skill
```

### 触发方式

安装后，当你在 Claude Code 中讨论以下话题时，Skill 会自动触发：

- 产品构思或新功能想法
- 产品方向的可行性验证
- 用户痛点分析
- 产品迭代决策
- Backlog 优先级排序
- 产品指标评估
- 产品复盘、迭代回顾

### 不会触发的场景

以下场景明确排除，避免误触发：

- 纯市场调研或竞品数据收集
- 技术架构选型
- UI/UX 视觉设计执行
- 项目排期和甘特图
- 销售或市场推广策略

### 对话流程

Skill 会按照以下流程引导对话（根据场景灵活调整，不一定每步都走）：

```
接住需求 → 痛点挖掘 → 极端推演 → 可行性判断 → 折中方案 → 验证路径 → 复盘回顾
```

### 输出模块

根据你的需求，Skill 会灵活组合以下输出模块：

| 模块 | 内容 |
|------|------|
| 痛点诊断 | 情感内核 + 行为证据 |
| 极端推演 | 版本 A / 版本 B / 推荐平衡点 |
| 可行性评估 | 用户价值 x 实现成本 x 市场时机 |
| 验证计划 | 最小实验 + 成功指标 + 时间框架 |
| 团队协作建议 | 谁需要参与、怎么对齐、沟通要点 |
| 复盘报告 | 假设验证表 + 意外发现 + 行动清单 |

## 使用示例

**你**：我想做一个微信小程序小商城，卖海参的

**Skill 的思考路径**：

1. 先不急着设计——"小商城"方向有赞/微店已经很成熟，直接做没有差异化
2. 挖痛点：海参是高客单价信任生意，通用电商工具为快消品设计，卖海参的人用着"不自如"
3. 极端推演：全功能电商平台 vs 继续微信群+收款码
4. 找到平衡点：不做"商城"，做"海参卖家成交助手"——先解决"每天重复讲解 30 遍"这个最痛的点
5. 给出 MVP 功能清单、技术方案、排期、成本估算、验证标准

完整产出见 [`examples/海参小程序落地方案.md`](examples/海参小程序落地方案.md)

## 自定义与扩展

### 修改触发条件

编辑 `agents/interface.yaml` 中的 `triggers` 部分，添加或移除触发/排除场景。

### 调整方法论

编辑 `references/methodology.md`，可以：
- 添加你自己团队的案例
- 修改复盘模板以匹配你的团队流程
- 增加新的方法论模块

### 调整语言风格

编辑 `SKILL.md` 中的"语言风格"部分，可以切换为更正式或更随意的沟通风格。
